您通过getUpdates或Webhook接收到的消息更新将包含特定消息的聊天ID，它将包含在message.chat.id关键字下。

这似乎是您能够检索聊天ID的唯一方法。所以，如果您想编写一个使机器人启动对话的程序，您可能需要将聊天ID与用户相关联并存储在某种键值存储中，如MemCache或Redis。

我认为他们的文档在这里建议了类似的做法，https://core.telegram.org/bots#deep-linking-example。您可以使用深度链接来启动对话，而无需要求用户首先输入一条消息。

使用Perl API，您可以通过以下方式获取：首先，您需要从Telegram向机器人发送一条消息，然后发出getUpdates请求，聊天ID必须在其中。 
#!/usr/bin/perl

use Data::Dumper;
use WWW::Telegram::BotAPI;

my $TOKEN = 'blablabla';
my $api = WWW::Telegram::BotAPI->new (
    token => $TOKEN
) or die "I can't connect";

my $out = $api->api_request ('getUpdates');
warn Dumper($out);
my $chat_id = $out->{result}->[0]->{message}->{chat}->{id};
print "chat_id=$chat_id\n";

id应该在chat_id中，但这可能取决于结果，因此我还添加了整个结果的转储。

您可以从https://github.com/Robertof/perl-www-telegram-botapi安装Perl API。它取决于您的系统，但我在我的Linux服务器上轻松运行了以下命令进行安装：

$ sudo cpan WWW::Telegram::BotAPI
希望这可以帮到您。

假设网站example.com希望通过Telegram机器人向其用户发送通知。以下是他们可以为ID为123的用户启用通知的方法。

  1. 创建一个适当的用户名为@ExampleComBot的机器人。
  2. 设置传入消息的Webhook。
  3. 生成一个足够长度的随机字符串，例如$ memcache_key =“vCH1vGWJxfSeofSAs0K5PA”。
  4. 将键$ memcache_key和值123放入Memcache中，有效期3600秒（一小时）。
  5. 向我们的用户展示按钮 https://telegram.me/ExampleComBot?start=vCH1vGWJxfSeofSAs0K5PA。
  6. 配置Webhook处理程序使用以/start开头的传入消息中传递的参数查询Memcached。 如果该键存在，则将聊天ID记录为telegram_chat_id，并与用户123相关联。从Memcache中删除该键。
  7. 现在当我们想要向用户123发送通知时，请检查是否有字段telegram_chat_id。如果有，则使用Bot API中的sendMessage方法向他们在Telegram中发送消息。

在扩展@Roberto Santalla的答案之上，如果您想要使用Telegram API与javascript和axios库一起使用，那么您可能需要以下内容：

const method = 'get'

const headers: any = {
  'Access-Control-Allow-Origin': '*',
  'Content-Type': 'application/json',
  timestamp: +new Date(),
}

const options = { headers: { ...headers } }

const urlTelegramBase =
  'https://api.telegram.org/bot123456:ABCDEF'

const urlGetUpdates = `${urlTelegramBase}/getUpdates`
const username = 'user_name'

const {
  data: { result: messages },
} = await axios[method](urlGetUpdates, options)

const chat_id = messages.find(
  messageBlock => messageBlock.message.chat.username === username
).message.chat.id

console.info('chat_id': chat_id)
